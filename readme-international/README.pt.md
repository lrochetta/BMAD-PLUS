## 💡 Por que BMAD+?

| BMAD-METHOD | BMAD+ |
|---|---|
| 9 agentes especializados | **5 agentes multi-função** (11 funções no total) |
| Ativação apenas manual | **Auto-ativação inteligente** em 3 níveis |
| Sem pipeline automatizado | **Modo Autopilot**: ideia → entrega |
| Execução sequencial | **Paralelismo supervisionado** |

## ⚡ Início rápido

```bash
npx bmad-plus install
```

### 💬 Com quem falar?

| Você quer... | Fale com | Exemplo |
|---|---|---|
| Discutir uma ideia de projeto | **Atlas** 🎯 | `Atlas, tenho uma ideia de projeto: um SaaS de faturamento` |
| Criar um PRD | **Atlas** 🎯 | `Atlas, crie o PRD para meu projeto` |
| Projetar a arquitetura | **Forge** 🏗️ | `Forge, proponha uma arquitetura` |
| Implementar código | **Forge** 🏗️ | `Forge, implemente a story AUTH-001` |
| Testar / code review | **Sentinel** 🔍 | `Sentinel, revise o módulo auth` |
| Planejar um sprint | **Nexus** 🎼 | `Nexus, crie epics e stories para o MVP` |
| Automatizar tudo | **Nexus** 🎼 | `autopilot` e descreva seu projeto |
| Investigação OSINT | **Shadow** 🔍 | `Shadow, investigate João Silva` |
| Criar um novo agente | **Maker** 🧬 | `Maker, crie um agente de suporte ao cliente` |

### 🔑 Comandos principais

| Comando | Descrição |
|---------|-------------|
| `bmad-help` | Ver todos os agentes e skills |
| `autopilot` | Nexus assume todo o pipeline |
| `parallel` | Execução multi-agente em paralelo |
