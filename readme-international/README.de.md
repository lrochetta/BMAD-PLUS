## 💡 Warum BMAD+?

| BMAD-METHOD | BMAD+ |
|---|---|
| 9 spezialisierte Agenten | **5 Multi-Rollen-Agenten** (11 Rollen gesamt) |
| Nur manuelle Aktivierung | **Intelligente Auto-Aktivierung** auf 3 Ebenen |
| Keine automatisierte Pipeline | **Autopilot-Modus**: Idee → Auslieferung |
| Sequenzielle Ausführung | **Überwachte Parallelisierung** |

## ⚡ Schnellstart

```bash
npx bmad-plus install
```

### 💬 Mit wem sprechen?

| Du willst... | Sprich mit | Beispiel |
|---|---|---|
| Eine Projektidee diskutieren | **Atlas** 🎯 | `Atlas, ich habe eine Projektidee: ein Rechnungs-SaaS` |
| Ein PRD erstellen | **Atlas** 🎯 | `Atlas, erstelle das PRD für mein Projekt` |
| Architektur entwerfen | **Forge** 🏗️ | `Forge, schlage eine Architektur vor` |
| Code implementieren | **Forge** 🏗️ | `Forge, implementiere Story AUTH-001` |
| Testen / Code-Review | **Sentinel** 🔍 | `Sentinel, überprüfe das Auth-Modul` |
| Sprint planen | **Nexus** 🎼 | `Nexus, erstelle Epics und Stories für das MVP` |
| Alles automatisieren | **Nexus** 🎼 | `autopilot` und beschreibe dein Projekt |
| OSINT-Recherche | **Shadow** 🔍 | `Shadow, investigate Max Müller` |
| Neuen Agenten erstellen | **Maker** 🧬 | `Maker, erstelle einen Kundensupport-Agenten` |

### 🔑 Wichtige Befehle

| Befehl | Beschreibung |
|--------|-------------|
| `bmad-help` | Alle Agenten und Skills anzeigen |
| `autopilot` | Nexus übernimmt die gesamte Pipeline |
| `parallel` | Multi-Agenten-Parallelausführung |
