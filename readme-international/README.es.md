## 💡 ¿Por qué BMAD+?

| BMAD-METHOD | BMAD+ |
|---|---|
| 9 agentes especializados | **5 agentes multi-rol** (11 roles en total) |
| Activación manual únicamente | **Auto-activación inteligente** en 3 niveles |
| Sin pipeline automatizado | **Modo Autopilot**: idea → entrega |
| Ejecución secuencial | **Paralelismo supervisado** |

## ⚡ Inicio rápido

```bash
npx bmad-plus install
```

### 💬 ¿Con quién hablar?

| Quieres... | Habla con | Ejemplo |
|---|---|---|
| Discutir una idea de proyecto | **Atlas** 🎯 | `Atlas, tengo una idea de proyecto: un SaaS de facturación` |
| Crear un PRD | **Atlas** 🎯 | `Atlas, crea el PRD para mi proyecto` |
| Diseñar la arquitectura | **Forge** 🏗️ | `Forge, propón una arquitectura` |
| Implementar código | **Forge** 🏗️ | `Forge, implementa la story AUTH-001` |
| Testear / code review | **Sentinel** 🔍 | `Sentinel, revisa el módulo auth` |
| Planificar un sprint | **Nexus** 🎼 | `Nexus, crea epics y stories para el MVP` |
| Automatizar todo | **Nexus** 🎼 | `autopilot` y describe tu proyecto |
| Investigación OSINT | **Shadow** 🔍 | `Shadow, investiga a Juan García` |
| Crear un nuevo agente | **Maker** 🧬 | `Maker, crea un agente de soporte al cliente` |

### 🔑 Comandos clave

| Comando | Descripción |
|---------|-------------|
| `bmad-help` | Ver todos los agentes y skills |
| `autopilot` | Nexus gestiona todo el pipeline |
| `parallel` | Ejecución multi-agente en paralelo |
