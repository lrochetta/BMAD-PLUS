## 💡 为什么选择 BMAD+？

| BMAD-METHOD | BMAD+ |
|---|---|
| 9个专业代理 | **5个多角色代理**（共11个角色） |
| 仅手动激活 | **3级智能自动激活** |
| 无自动化流水线 | **自动驾驶模式**：想法 → 交付 |
| 顺序执行 | **受监督的并行执行** |

## ⚡ 快速开始

```bash
npx bmad-plus install
```

### 💬 和谁交谈？

| 你想要... | 找 | 示例 |
|---|---|---|
| 讨论项目想法 | **Atlas** 🎯 | `Atlas, 我有一个项目想法：一个发票SaaS` |
| 创建PRD | **Atlas** 🎯 | `Atlas, 为我的项目创建PRD` |
| 设计架构 | **Forge** 🏗️ | `Forge, 提出一个架构方案` |
| 实现代码 | **Forge** 🏗️ | `Forge, 实现故事 AUTH-001` |
| 测试/代码审查 | **Sentinel** 🔍 | `Sentinel, 审查auth模块` |
| 规划冲刺 | **Nexus** 🎼 | `Nexus, 为MVP创建史诗和故事` |
| 全部自动化 | **Nexus** 🎼 | `autopilot` 然后描述你的项目 |
| OSINT调查 | **Shadow** 🔍 | `Shadow, investigate 张三` |
| 创建新代理 | **Maker** 🧬 | `Maker, 创建一个客户支持代理` |

### 🔑 关键命令

| 命令 | 描述 |
|------|------|
| `bmad-help` | 显示所有代理和技能 |
| `autopilot` | Nexus接管整个流水线 |
| `parallel` | 启动多代理并行执行 |
